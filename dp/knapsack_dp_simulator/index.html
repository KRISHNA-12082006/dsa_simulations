0/1 Knapsack – Visual DP Simulator

Question — Samasya Kya Hai?

Humare paas n items hain. Har item ka ek weight (wi) aur ek value (vi) hai. Saath hi, hamare paas ek knapsack (bag) hai jiski capacity W hai — matlab usmein hum jitna weight daal sakte hain, uski limit W hai.

Hume decide karna hai ki kaunse items knapsack mein daalein, taki total weight W se zyada na ho aur total value jitni ho sake, wo maximize ho jaye. Aur ek item ko ya to poora lena hai ya bilkul nahi (fractional nahi — isliye “0/1”).

Udāharan ke liye: agar weights = [2, 3, 4, 5] aur values = [3, 4, 5, 8], aur capacity W = 10, to hum ye dekhna chaheinge ki kaunse items lene se value sabse zyada hogi jab weight ki constraint bhi ho.

Approach — Dynamic Programming (DP)

Ye problem hum dynamic programming (DP) se bahut achhi tarah solve kar sakte hain, kyunki iska "optimal substructure" aur "overlapping subproblems" dono hain:

  • State Definition: Let dp[i][w] = maximum value hum paa sakte hain agar hum pehle i items ko dekhein aur remaining capacity exactly w ho.
  • Recurrence (DP Relation):
    • Agar weight of current item \(w_i\) > w → matlab hum us item ko nahi le sakte: \[ dp[i][w] = dp[i-1][w] \]
    • Warna (agar le sakte hain) → do choices hain: 1. **Include karna:** value = \(v_i + dp[i-1][w - w_i]\) 2. **Exclude karna:** value = \(dp[i-1][w]\) To hum choose karenge unmein se jo zyada value de raha ho: \[ dp[i][w] = \max(dp[i-1][w], v_i + dp[i-1][w - w_i]) \]
  • Base Case: - Agar \(i = 0\) (koi item nahi), to dp[0][w] = 0 for any w. - Agar \(w = 0\) (capacity 0), to dp[i][0] = 0 for any i.

Time Complexity: Agar n = number of items aur W = capacity, to bottom-up tabulation mein time complexity hoti hai \(O(n \times W)\). Yeh ek pseudo-polynomial time algorithm hai. :contentReference[oaicite:0]{index=0}

Kyu DP karna sahi hai (Greedy kyun fail karegi)? Agar hum greedy strategy (jaise sabse zyada value-per-weight chunna) use karte hain, to wo hamesha optimal nahi hoti. 0/1 knapsack mein ek example ho sakta hai jahan value-per-weight zyada hai par wo item lene se doosre better combination garbar ho sakta hai. :contentReference[oaicite:1]{index=1}
